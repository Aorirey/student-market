# Отчёт по безопасности OWASP Top 10

## Анализ и устранение уязвимостей OWASP Top 10 2021

Этот документ описывает меры безопасности, реализованные для защиты маркетплейса учебных работ от уязвимостей OWASP Top 10.

---

## A01:2021 — Broken Access Control (Нарушение контроля доступа)

### Выявленные проблемы:
- ❌ Прямая конкатенация пользовательских данных в SQL-запросах
- ❌ Отсутствие проверки прав доступа к административным функциям

### Реализованные меры:
✅ **Параметризованные запросы** — все SQL-запросы используют параметризованный синтаксис (`?` для SQLite, `$1, $2...` для PostgreSQL)
✅ **Валидация прав доступа** — проверка `currentUser.isAdmin` перед доступом к админ-панели
✅ **Санитизация выходных данных** — функция `sanitizeHTML()` для всех данных из БД

### Пример безопасного кода:
```javascript
// БЫЛО (уязвимо):
db.exec(`SELECT * FROM users WHERE id = '${req.params.id}'`)

// СТАЛО (безопасно):
db.exec("SELECT * FROM users WHERE id = ?", [req.params.id])
```

## A02:2021 — Cryptographic Failures (Криптографические ошибки)

### Выявленные проблемы:
- ❌ Пароли хранились в открытом виде
- ❌ Отсутствие хеширования чувствительных данных

### Реализованные меры:
✅ **Хеширование паролей** — используется `bcrypt` с солью (10 раундов)
✅ **Безопасное сравнение паролей** — `bcrypt.compare()` для проверки
✅ **Установка пакетов**: `npm install bcrypt --save`

### Пример:
```javascript
// Регистрация:
const hashedPassword = await bcrypt.hash(password, 10);

// Вход:
const isValidPassword = await bcrypt.compare(password, hashedPassword);
```

## A03:2021 — Injection (Инъекции)

### Выявленные проблемы:
- ❌ SQL-инъекции через конкатенацию строк
- ❌ Отсутствие валидации входных данных

### Реализованные меры:
✅ **Параметризованные запросы** — во всех SQL-запросах
✅ **Валидация входных данных** — используется `express-validator`
✅ **Ограничение длины данных** — максимальная длина для всех полей
✅ **Валидация типов** — проверка числовых значений, email, дат

### Пример валидатора:
```javascript
const registerValidator = [
    body('name').trim().notEmpty().isLength({ max: 100 }),
    body('email').trim().isEmail().normalizeEmail(),
    body('password').isLength({ min: 6, max: 128 }),
    validate
];
```

## A04:2021 — Insecure Design (Небезопасный дизайн)

### Выявленные проблемы:
- ❌ Отсутствие ограничений на количество запросов
- ❌ Нет валидации размеров файлов
- ❌ Отсутствие ограничений на размер данных

### Реализованные меры:
✅ **Rate Limiting** — ограничение запросов через `express-rate-limit`:
  - 100 запросов за 15 минут (общий)
  - 5 попыток входа за 15 минут (аутентификация)
  - 10 запросов за час (строгий лимит)

✅ **Валидация файлов**:
  - Максимальный размер: 10MB
  - Разрешённые расширения: pdf, doc, docx, png, jpg, jpeg, zip, rar
  - Валидация Base64 для данных файлов

✅ **Ограничения данных**:
  - Максимальная длина пароля: 128 символов
  - Максимальная цена: 1,000,000 ₽
  - Максимальный размер JSON: 10MB

## A05:2021 — Security Misconfiguration (Ошибки конфигурации)

### Выявленные проблемы:
- ❌ Отсутствие заголовков безопасности
- ❌ CORS разрешён для всех источников
- ❌ Нет ограничений на методы HTTP

### Реализованные меры:
✅ **Helmet.js** — заголовки безопасности:
  - Content Security Policy (CSP)
  - X-Content-Type-Options
  - X-Frame-Options
  - X-XSS-Protection
  - Strict-Transport-Security

✅ **Настроенный CORS**:
  - Ограничение по origin (через `ALLOWED_ORIGINS`)
  - Разрешены только необходимые методы
  - Ограниченные заголовки

### Пример:
```javascript
app.use(helmet({
    contentSecurityPolicy: {
        directives: {
            defaultSrc: ["'self'"],
            scriptSrc: ["'self'", "'unsafe-inline'"],
            imgSrc: ["'self'", 'data:', 'blob:']
        }
    }
}));
```

## A07:2021 — Identification and Authentication Failures

### Выявленные проблемы:
- ❌ Нет защиты от перебора паролей
- ❌ Слабые требования к паролям
- ❌ Сессии в sessionStorage (уязвимо для XSS)

### Реализованные меры:
✅ **Rate Limiting** — 5 попыток входа за 15 минут
✅ **Требования к паролям** — минимум 6 символов
✅ **Хеширование паролей** — bcrypt с солью
✅ **Блокировка аккаунтов** — через `isBlocked` флаг

### Рекомендации для production:
⚠️ Рассмотрите использование JWT токенов с коротким временем жизни
⚠️ Добавьте двухфакторную аутентификацию (2FA)
⚠️ Реализуйте refresh token механизм

## A08:2021 — Software and Data Integrity Failures

### Выявленные проблемы:
- ❌ XSS через `innerHTML` с пользовательскими данными
- ❌ Отсутствие валидации типов файлов
- ❌ Нет проверки целостности данных

### Реализованные меры:
✅ **XSS защита**:
  - Функция `escapeHTML()` для экранирования
  - Замена `innerHTML` на `textContent` где возможно
  - Санитизация всех данных из БД

✅ **Валидация файлов**:
  - Проверка расширения файла
  - Валидация Base64 данных
  - Ограничение размера (10MB)

✅ **Целостность данных**:
  - Валидация числовых значений
  - Проверка диапазонов (цена, баланс, рейтинг)

### Пример XSS защиты:
```javascript
// БЫЛО (уязвимо):
card.innerHTML = `<h3>${item.title}</h3>`;

// СТАЛО (безопасно):
const titleEl = document.createElement('h3');
titleEl.textContent = item.title;
```

## A09:2021 — Security Logging and Monitoring Failures

### Выявленные проблемы:
- ❌ Отсутствие логирования событий безопасности
- ❌ Нет мониторинга подозрительной активности

### Реализованные меры:
✅ **Логирование событий**:
  - `[AUTH]` — регистрация, вход, неудачные попытки
  - `[AUDIT]` — удаление пользователей
  - `[MODERATION]` — одобрение/отклонение товаров
  - `[PURCHASE]` — создание покупок
  - `[FILE]` — загрузка файлов

### Пример:
```javascript
console.log(`[AUTH] Неудачная попытка входа: ${email}`);
console.log(`[AUDIT] Пользователь удален: ${req.params.id}`);
```

### Рекомендации для production:
⚠️ Используйте Winston или Pino для структурированного логирования
⚠️ Настройте алерты на подозрительную активность
⚠️ Реализуйте SIEM интеграцию

## Дополнительные меры безопасности

### 1. Валидация на клиенте
```javascript
function isValidEmail(email) {
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return emailRegex.test(email);
}

function isValidNumber(value, min = 1, max = 1000000) {
    const num = typeof value === 'string' ? parseInt(value, 10) : value;
    return !isNaN(num) && num >= min && num <= max;
}
```

### 2. Безопасная работа с сессиями
```javascript
// Сохранение пользователя
sessionStorage.setItem('currentUser', JSON.stringify(currentUser));

// Восстановление сессии
const savedUser = sessionStorage.getItem('currentUser');
if (savedUser) {
    currentUser = JSON.parse(savedUser);
}
```

### 3. Конфигурация для production
```bash
# .env production
NODE_ENV=production
ALLOWED_ORIGINS=https://yourdomain.com
DATABASE_URL=postgresql://secure-connection
PORT=3000
```
