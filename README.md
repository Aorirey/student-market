# 🎓 StudentMarket — Маркетплейс учебных работ

Платформа для покупки и продажи учебных работ (практические, лабораторные, курсовые).

Следуйте этой инструкции, чтобы разместить сайт **бесплатно** и сделать его доступным для всех.

## 📝 Шаг 1: Загрузка кода на GitHub

### 1.1 Создайте репозиторий на GitHub

1. Зайдите на [github.com](https://github.com)
2. Войдите в свой аккаунт (или зарегистрируйтесь)
3. Нажмите **"+"** → **"New repository"**
4. Назовите репозиторий, например: `student-market`
5. Сделайте репозиторий **Public**
6. Нажмите **"Create repository"**

### 1.2 Загрузите файлы проекта

Откройте PowerShell в папке проекта и выполните:

```powershell
# Инициализация Git
git init

# Добавляем все файлы
git add .

# Первое сохранение
git commit -m "Initial commit"

# Добавляем GitHub как удалённый репозиторий
# Замените YOUR_USERNAME на ваш логин GitHub
git remote add origin https://github.com/YOUR_USERNAME/student-market.git

# Отправляем код на GitHub
git branch -M main
git push -u origin main
```

## 🗄️ Шаг 2: Создание базы данных на Supabase

### 2.1 Регистрация на Supabase

1. Перейдите на [supabase.com](https://supabase.com)
2. Нажмите **"Start your project"**
3. Войдите через GitHub (рекомендуется) или через email

### 2.2 Создание проекта

1. Нажмите **"New project"**
2. Заполните:
   - **Name:** `studentmarket`
   - **Database password:** придумайте надёжный пароль (запомните его!)
   - **Region:** выберите ближайший (например, `Frankfurt (germany)`)
3. Нажмите **"Create new project"**
4. Дождитесь создания (2-3 минуты)

### 2.3 Получение строки подключения

1. В левом меню нажмите **Settings** (шестерёнка) → **Database**
2. Найдите раздел **"Connection string"**
3. Переключитесь на вкладку **URI**
4. Скопируйте строку подключения
5. **Замените `[YOUR-PASSWORD]`** на ваш пароль, который вы придумали

Строка будет выглядеть так:
```
postgresql://postgres.xxxxx:ВАШ_ПАРОЛЬ@aws-0-region.pooler.supabase.com:6543/postgres
```

> ⚠️ **Важно:** Не забудьте заменить `[YOUR-PASSWORD]` на реальный пароль!

## 🖥️ Шаг 3: Размещение на Render

### 3.1 Регистрация на Render

1. Перейдите на [render.com](https://render.com)
2. Нажмите **"Get Started for Free"**
3. Войдите через GitHub

### 3.2 Создание сервиса

1. Нажмите **"New +"** → **"Web Service"**
2. Нажмите **"Connect a repository"**
3. Выберите ваш репозиторий `student-market`
4. Заполните настройки:

| Поле | Значение |
|------|----------|
| **Name** | `student-market` |
| **Region** | Frankfurt, Germany (или ближайший) |
| **Branch** | `main` |
| **Root Directory** | (оставьте пустым) |
| **Runtime** | `Node` |
| **Build Command** | `npm install` |
| **Start Command** | `node server.js` |
| **Instance Type** | `Free` |

### 3.3 Добавление переменных окружения

Нажмите **"Advanced"** → **"Add Environment Variable"** и добавьте:

```
Name: DB_MODE
Value: postgres
```

```
Name: DATABASE_URL
Value: (ваша строка подключения от Neon из шага 2.3)
```

```
Name: NODE_ENV
Value: production
```

### 3.4 Запуск

1. Нажмите **"Create Web Service"**
2. Начнётся сборка (около 2-5 минут)
3. Когда появится зелёная галочка — сервер запущен!
4. Ваш сайт доступен по адресу: `https://student-market-xxxx.onrender.com`

## ⚠️ Важно!

### Для локальной разработки
Используется **SQLite** (файл `database.sqlite`). Все данные хранятся локально.

### Для продакшена (Render)
Используется **PostgreSQL** (Neon). Данные хранятся в облаке и не теряются.

Переключение происходит автоматически через переменную `DB_MODE`:
- `DB_MODE=sqlite` — локальный режим
- `DB_MODE=postgres` — продакшен режим

## 🆘 Решение проблем

### Ошибка: "DATABASE_URL не указан"

Добавьте переменную окружения `DATABASE_URL` в настройках Render с вашей строкой подключения от Supabase.

### Ошибка: "Cannot find module 'pg'"

Выполните локально:
```powershell
npm install
git add .
git commit -m "Add pg dependency"
git push
```

### Сервер на Render "засыпает"

На бесплатном тарифе Render сервер "засыпает" после 15 минут бездействия. Первый запрос после простоя будет обрабатываться ~30 секунд. Это нормально.

### Ошибка подключения к базе данных

1. Проверьте, что строка подключения правильная
2. Убедитесь, что заменили `[YOUR-PASSWORD]` на реальный пароль
3. Проверьте, что `DB_MODE=postgres` установлена
4. Перезапустите сервер на Render (кнопка "Restart")
